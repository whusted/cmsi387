Symbol Description
.
 .The "dot" command (or period) refers to the current directory the user is in.
&
 This command means "Run job in background." A command followed by an & will run in the background.
<
 This symbol is used to redirect input.
|
 The "pipe" symbol passes the output (stdout) of a previous command to the input (stdin) of the next one, or to the shell. This is a method of chaining commands together.
>>
 This symbol appends the output of one file to another (possibly new) file.
?
 "?" is a test operator. Within certain expressions, the ? indicates a test for a condition.
;
 A command separator that permits putting two or more commands on the same line.
*
 The * character serves as a "wild card" for filename expansion. By itself, it matches every filename in a given directory.
'
Using single quotes (' ... ') causes the variable name to be used literally, and no substitution will take place. This is full quoting, sometimes referred to as 'strong quoting.'
#
 Lines beginning with a # are comments and will not be executed.
{}
 Also referred to as an inline group, this construct, in effect, creates an anonymous function (a function without a name). However, unlike in a "standard" function, the variables inside a code block remain visible to the remainder of the script.
