Unix Command Crib Sheet

Click on a command to read more about it.

Command Example
cd cd homework/shell-guide
ls ls
mkdir mkdir new_folder
cat cat example.txt anotherexample.txt
ssh ssh whusted@my.cs.lmu.edu
top top
kill kill 1111
grep grep willy testfile.txt
nice nice -20 test.java
sudo sudo rm test.txt
rm rm test.txt

Short for "Change directory", cd (optionally) takes one argument: the name of the desired directory the user wishes to navigate to. cd alone will take the user to his or her home directory. In the example, the user is switching to his or her homework/shell-guide directory.

Short for "List", ls lists all the files in the user's current directory. Optional arguments are available, such as "-l", which lists files in "long format" (size, date modified, etc.).

Short for "make directory", mkdir does just that. The required argument after mkdir is the name of the new directory. This command is the same as creating a new folder in a Graphical User Interface. In the example, a new directory would be created called "new_folder".

Short for "Catenate", cat takes at least one argument (in the form of a text file) and concatenates the file(s) and prints to standard output. In the example, cat would concatenate the two files provided as arguments.

Short for "Secure Shell", ssh is a progam for logging into a remote machine and for executing commands on a remote machine. LMU students will most likely use ssh in the above example. In the example, I am ssh'ing into my Keck lab account.

Short for "Table of Processes", the top command displays to standard output an ordered list of running processes, updating periodically if left running. Pragmatically it can be used to see if a certain application is using a lot of the CPU.

"Kill" is a command that is used in several popular operating systems to send signals to running processes in order to request the termination of the process. The argument is the process's ID. Like many other commands, this one does what it sounds like it does. In the example, process number 1111 will be terminated.

"Grep" is a utility for searching plain-text data sets for lines matching a regular expression. In the example above, "willy" is the regular expression being searched for in testfile.txt. If found, the expression is highlighted and printed to standard output. If not found, nothing is printed to standard output.

"nice" is used to invoke a utility or shell script with a particular priority, thus giving the process more or less CPU time than other processes. -20 (used in the above example) is the highest priority a process can have. In the example above, the process test.java is given the highest priority.

sudo is a program for Unix-like computer operating systems that allows users to run programs with the security privileges of another user (usually this is the superuser). sudo requires a password to be performed; it's like a Trump card. In the example, I wish to remove the file test.txt but require superuser authority to do so.

Short for "remove", rm removes the file whose name is given as the argument. In the above example, test.txt is removed from the current directory (if it is in fact present). In the example, the file test.txt is removed.
